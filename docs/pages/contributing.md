# Contributing to PortBay

PortBay Community is open source and welcomes contributions. Whether you're fixing a bug, improving documentation, adding a recipe, or proposing a new feature — there's a place for you here.

---

## Earn a Perpetual Pro License

This is worth calling out clearly: **a merged qualifying pull request earns you a perpetual PortBay Pro license**, tied to your GitHub account. No subscription. No renewal. It is our way of acknowledging that contributors make this project better for everyone.

The same applies to sponsors: a qualifying donation through [GitHub Sponsors](https://github.com/sponsors/portbay-app) also earns a perpetual Pro license.

What counts as "qualifying" is documented in the repository. In general: any meaningful code or documentation contribution that improves the project for other users is a candidate. Trivial one-character fixes or automated PRs do not qualify.

---

## How to Start

**1. Browse good first issues**  
Issues labeled [`good first issue`](https://github.com/portbay-app/portbay/issues?q=is%3Aopen+label%3A%22good+first+issue%22) are the intended entry point. They are scoped to be approachable without deep knowledge of the codebase.

**2. Join Discussions**  
[GitHub Discussions](https://github.com/portbay-app/portbay/discussions) is where feature proposals, architecture questions, and general conversation happen. If you are unsure whether your idea fits, start there before opening a PR.

**3. Read the contributing guides**  
[`CONTRIBUTING.md`](../../CONTRIBUTING.md) at the repo root covers the mechanics: how to set up a dev environment, branch naming, PR expectations, and the review process. The guides under [`docs/contributing/`](../contributing/) go deeper on specific topics.

## Developer Certificate of Origin (DCO)

PortBay uses the DCO instead of a Contributor License Agreement. When you submit a pull request, each of your commits should be signed off to indicate that you wrote the code and have the right to contribute it under the project license.

Add the sign-off with:

```
git commit -s -m "your commit message"
```

That adds a `Signed-off-by: Your Name <your@email.com>` trailer to the commit. It is a lightweight statement — not a legal agreement — that you are contributing in good faith. See the [DCO](https://developercertificate.org/) for the full text.

## What's Welcome

- Bug fixes (with a test that demonstrates the fix, if practical)
- Performance improvements (with before/after evidence)
- Documentation improvements — clarity, accuracy, coverage
- New project recipes
- UI and UX polish
- Translations (check Discussions for the current localization status)
- Accessibility improvements

---

## What Needs Discussion First

Some changes have broad impact and benefit from alignment before implementation:

- New top-level features or significant UI redesigns
- Changes to the data model, configuration format, or recipe schema
- Changes that affect how PortBay interacts with the OS (hosts file, certificate trust, launchd, etc.)
- Changes to the plugin interface or CLI surface

Open a Discussion or comment on an existing issue before investing time in a large PR. This protects your time and makes the review process smoother.
