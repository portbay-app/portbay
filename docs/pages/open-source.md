# PortBay is Open Source

PortBay Community is the open-source local development manager for individuals and teams who want a clean, transparent way to run projects locally.

It manages local projects, ports, services, local `.test` domains, local HTTPS, reverse-proxy routing, project start/stop, logs, and metrics — all on your machine, without containers or cloud dependencies. The Community edition is free, fully offline-capable, and always will be.

---

## What's in Community (free, AGPL-3.0)

Everything you need to manage a local development environment on macOS today (Apple Silicon), with Intel Mac, Linux, and Windows support planned:

- Local project manager — start, stop, and monitor any project
- Port and service management
- Local `.test` domain routing
- Local HTTPS via mkcert-based certificate trust
- Reverse-proxy routing
- Logs and metrics
- Basic project recipes
- CLI and menu-bar mode
- Open plugin interfaces
- Import from Herd, MAMP, ServBay

Community is a complete, standalone tool. No account required. No telemetry without consent. No features gated behind a paywall.

## What's in Cloud / Pro (commercial, separate)

PortBay Cloud and Pro features are developed separately and may include team sync, cloud backups, remote access, hosted recipes, billing, organization management, enterprise policy controls, and managed infrastructure.

These are optional and additive. Nothing in Community is crippled to upsell Pro.

See the full comparison → [Community vs. Pro](./community-vs-pro.md)

## Why Open Source

**Transparency.** A local development tool runs on your machine, with your credentials, your project paths, and your environment variables. You should be able to read the code and verify exactly what it does.

**Local-first.** PortBay is built to work offline and to keep your data on your machine. Open source reinforces that: the tool cannot become a cloud product behind your back.

**No lock-in.** You can fork it, build it yourself, inspect its behavior, or switch away. The source is there.

**Community input.** The people who use a developer tool daily usually know what it needs next. Open development keeps the roadmap honest.
