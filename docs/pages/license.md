## What AGPL-3.0 means in practice

The AGPL is a copyleft license in the GPL family with one additional condition: the network use clause.

**You can:**

- Use PortBay Community for any purpose — personal, commercial, internal
- Study the source code
- Modify it for your own use
- Share unmodified or modified copies

**You must (when distributing or deploying over a network):**

- Make the corresponding source code available under AGPL-3.0 terms
- Keep copyright and license notices intact
- Pass on the same freedoms to recipients

The key AGPL addition over GPL: if you run a **modified version of PortBay as a service** — meaning users interact with it over a network — you must provide the modified source under AGPL terms. Running PortBay locally and unmodified (the normal case) has no such requirement.

---

## Why AGPL?

We chose AGPL-3.0 to keep the community edition and any network-served derivatives open. A permissive license would let someone take the tool, modify it, offer it as a hosted service, and never contribute the changes back. AGPL closes that gap.

It does not affect ordinary use. Developers using PortBay Community to manage their local projects are not distributing software or running a network service — AGPL places no additional obligations on them.

---

## What about PortBay Cloud and Pro?

PortBay Cloud and Pro are **separate commercial software and services**, not covered by the AGPL. They are developed in a private repository. The AGPL does not require PortBay's proprietary backend services to open-source code that merely communicates with the Community app over public APIs.

In other words: the open-source app can have a commercial companion without violating the intent of the AGPL, as long as the companion is genuinely separate software — which it is.
