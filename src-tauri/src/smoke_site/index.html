What it does

Your machine, treated like a tiny PaaS

Each project is a declarative record — a hostname, a start command, a port. PortBay owns the rest: lifecycle, routing, and certificates.

One-click Play / Stop

Start and stop Next.js, Vite, Node, PHP, or Laravel per project — plus a universal Stop-All that always works, even after a crash.

Real HTTPS on .test

Locally-trusted hostnames like myproject.test, issued and trusted on your machine via mkcert.

Wildcard .test routing

A bundled DNS resolver answers every *.test — no edits to /etc/hosts, no proxy config to maintain.

Bundled databases

PortBay-supervised MySQL, MariaDB, Postgres, and Redis — started and stopped alongside the projects that need them.

Public sharing

Expose any project over a Cloudflare tunnel with one click — a shareable URL for the demo, the webhook, or the phone on your desk.

Full-parity CLI

A Rust core with complete CLI parity — the GUI is a client, not the source of truth. Script it, drive it from an agent, or live in the menu bar.

FAQ

Questions, answered

Still curious? The docs go deeper.

Do I need Docker?

No. PortBay supervises native processes directly — no containers, no VM, no engine to keep running. That's why idle RAM stays under 80 MB.

Is it really free?

Yes. PortBay Community is free and open source under AGPL-3.0. It's not a trial and nothing is crippled to push you to Pro — Community runs fully offline with no account.

Which frameworks and runtimes?

Next.js, Vite, plain Node, PHP and Laravel, and static sites. Each project declares its own start command, so anything you can launch from a terminal works.

How do I get HTTPS on .test?

PortBay uses a local mkcert certificate authority to issue and trust certificates on your machine, and routes traffic through a bundled Caddy reverse proxy. No self-signed cert warnings.

Can I import from Herd, MAMP or ServBay?

Yes. PortBay can import existing sites from Laravel Herd, MAMP and ServBay so you don't have to re-register every project by hand.

Does it edit my /etc/hosts file?

Wildcard *.test resolution is handled by a bundled DNS resolver, so you don't hand-edit hosts per project. Any managed entries go through an auditable privileged helper.

How do I get Pro?

Two ways, both granting the same perpetual license: have a meaningful pull request merged, or sponsor the project. No subscription, no renewal.

Which platforms are supported?

macOS (Apple Silicon) ships today as a signed, notarized build. Linux and Windows are on the roadmap.
